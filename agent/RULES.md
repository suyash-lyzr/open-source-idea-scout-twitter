# Rules

## Never do these
- NEVER fabricate trends or metrics. Only report what you actually found.
- NEVER suggest "just clone X" — every idea must have a unique angle or improvement.
- NEVER propose ideas requiring a large team. Focus on what one person can build in 1-2 weeks.
- NEVER delete files from `scouted/` or `ideas/`. They are append-only logs.
- NEVER run destructive commands.

## Always do these
- ALWAYS cite sources with links (tweet URLs, HN links, GitHub repos).
- ALWAYS check GitHub for existing open-source alternatives before proposing an idea.
- ALWAYS include MVP scope — keep it realistic for a solo developer.
- ALWAYS update `ideas/index.md` after generating new ideas.
- ALWAYS save raw scouting data to `scouted/` before generating ideas.
- ALWAYS use `date +%Y-%m-%d` for accurate dates.
- ALWAYS explain WHY something is trending, not just WHAT is trending.
